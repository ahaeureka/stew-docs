# 用户管理接口可配置化方案

> 支持灵活接入第三方用户管理平台（Casdoor、Keycloak、Auth0 等）

---

## 一、设计目标

将用户管理操作（获取用户信息、更新用户信息等）抽象为接口（trait），通过配置选择不同的实现。

---

## 二、架构设计

### 2.1 核心抽象

```rust
/// 用户管理提供者 trait
/// 抽象不同第三方平台的用户管理操作
pub trait UserManagementProvider: Send + Sync {
    /// 获取用户信息
    async fn get_user(&self, access_token: &str) -> Result<UserInfo, AuthError>;

    /// 更新用户信息
    async fn update_user(
        &self,
        access_token: &str,
        update: &UserUpdateRequest,
    ) -> Result<UserInfo, AuthError>;

    /// 上传头像
    async fn upload_avatar(
        &self,
        access_token: &str,
        avatar_data: &[u8],
        filename: &str,
    ) -> Result<String, AuthError>;
}
```

### 2.2 配置驱动

```yaml
# config.yaml
user_management:
  # 提供者类型: casdoor | keycloak | auth0 | custom
  provider: "casdoor"

  # Casdoor 配置
  casdoor:
    endpoint: "https://door.casdoor.com"
    # 可选：自定义 API 路径
    # api_path: "/api"

  # Keycloak 配置
  keycloak:
    endpoint: "https://keycloak.example.com"
    realm: "my-realm"

  # Auth0 配置
  auth0:
    domain: "mytenant.auth0.com"
```

### 2.3 实现类

| 实现 | 说明 |
|------|------|
| `CasdoorUserProvider` | Casdoor 用户管理 |
| `KeycloakUserProvider` | Keycloak 用户管理 |
| `Auth0UserProvider` | Auth0 用户管理 |
| `MockUserProvider` | 开发测试用 |

### 3.3 Casdoor 实现示例

```rust
pub struct CasdoorUserProvider {
    client: reqwest::Client,
    base_url: String,
}

impl CasdoorUserProvider {
    pub fn new(base_url: String) -> Self {
        Self {
            client: reqwest::Client::new(),
            base_url,
        }
    }
}

#[async_trait]
impl UserManagementProvider for CasdoorUserProvider {
    async fn get_user(&self, access_token: &str) -> Result<UserInfo, AuthError> {
        let response = self.client
            .get(format!("{}/api/userinfo", self.base_url))
            .header("Authorization", format!("Bearer {}", access_token))
            .send()
            .await?;

        let user_info: UserInfo = response.json().await?;
        Ok(user_info)
    }

    async fn update_user(
        &self,
        access_token: &str,
        update: &UserUpdateRequest,
    ) -> Result<UserInfo, AuthError> {
        let response = self.client
            .post(format!("{}/api/update-user", self.base_url))
            .header("Authorization", format!("Bearer {}", access_token))
            .json(update)
            .send()
            .await?;

        let user_info: UserInfo = response.json().await?;
        Ok(user_info)
    }

    async fn upload_avatar(
        &self,
        access_token: &str,
        avatar_data: &[u8],
        filename: &str,
    ) -> Result<String, AuthError> {
        let form = reqwest::multipart::Form::new()
            .part("file", reqwest::multipart::Part::bytes(avatar_data.to_vec())
                .file_name(filename.to_string()));

        let response = self.client
            .post(format!("{}/api/upload-avatar", self.base_url))
            .header("Authorization", format!("Bearer {}", access_token))
            .multipart(form)
            .send()
            .await?;

        let result: serde_json::Value = response.json().await?;
        let avatar_url = result["data"].as_str().ok_or_else(|| {
            AuthError::InternalError("Failed to get avatar URL".to_string())
        })?;

        Ok(avatar_url.to_string())
    }
}
```

### 3.4 Keycloak 实现示例

```rust
pub struct KeycloakUserProvider {
    client: reqwest::Client,
    base_url: String,
    realm: String,
}

impl KeycloakUserProvider {
    pub fn new(base_url: String, realm: String) -> Self {
        Self { client: reqwest::Client::new(), base_url, realm }
    }
}

#[async_trait]
impl UserManagementProvider for KeycloakUserProvider {
    async fn get_user(&self, access_token: &str) -> Result<UserInfo, AuthError> {
        // Keycloak 使用 /protocol/openid-connect/userinfo
        let response = self.client
            .get(format!(
                "{}/realms/{}/protocol/openid-connect/userinfo",
                self.base_url, self.realm
            ))
            .header("Authorization", format!("Bearer {}", access_token))
            .send()
            .await?;

        let user_info: UserInfo = response.json().await?;
        Ok(user_info)
    }

    async fn update_user(
        &self,
        access_token: &str,
        update: &UserUpdateRequest,
    ) -> Result<UserInfo, AuthError> {
        // Keycloak 使用 /admin/realms/{realm}/users/{id}
        // 需要先从 token 获取用户 ID
        let user_id = self.get_user_id_from_token(access_token).await?;

        let mut user_repr = serde_json::Map::new();
        if let Some(v) = &update.display_name {
            user_repr.insert("displayName".to_string(), serde_json::json!(v));
        }
        // ... 其他字段映射

        let response = self.client
            .put(format!(
                "{}/admin/realms/{}/users/{}",
                self.base_url, self.realm, user_id
            ))
            .header("Authorization", format!("Bearer {}", access_token))
            .json(&user_repr)
            .send()
            .await?;

        // Keycloak 更新不返回用户，需要重新获取
        self.get_user(access_token).await
    }
}
```

```rust
pub struct UserManagementConfig {

    /// Auth0 配置

    pub auth0: Option<Auth0ProviderConfig>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CasdoorProviderConfig {
    pub endpoint: String,
    #[serde(default)]
    pub api_path: Option<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct KeycloakProviderConfig {
    pub endpoint: String,
    pub realm: String,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Auth0ProviderConfig {
    pub domain: String,
}
```

### 4.2 用户服务改造

**文件**: `rustauth/src/services/user.rs`

```rust
pub struct UserServiceImpl {
    oidc_config: Arc<OidcConfig>,
    token_validator: StandardTokenValidator,
    /// 用户管理提供者
    user_provider: Arc<dyn UserManagementProvider>,
}

impl UserServiceImpl {
    pub fn new(
        oidc_config: Arc<OidcConfig>,
        user_provider: Arc<dyn UserManagementProvider>,
    ) -> Self {
        let token_validator = StandardTokenValidator::new(oidc_config.clone());
        Self {
            oidc_config,
            token_validator,
            user_provider,
        }
    }

    /// 更新用户信息
    async fn update_user(
        &self,
        token: &str,
        req: &UpdateUserRequest,
    ) -> Result<User, Status> {
        // 调用可配置的用户管理提供者
        let updated_info = self.user_provider
            .update_user(token, req)
            .await
            .map_err(|e| Status::internal(e.to_string()))?;

        Ok(self.user_info_to_proto(&updated_info))
    }
}
```

## 五、初始化工厂

```rust
impl UserManagementConfig {
    /// 根据配置创建用户管理提供者
    pub fn create_provider(&self) -> Arc<dyn UserManagementProvider> {
        match self.provider.as_str() {
            "casdoor" => {
                let config = self.casdoor.as_ref()
                    .expect("Casdoor config required");
                Arc::new(CasdoorUserProvider::new(config.endpoint.clone()))
            }
            "keycloak" => {
                let config = self.keycloak.as_ref()
                    .expect("Keycloak config required");
                Arc::new(KeycloakUserProvider::new(
                    config.endpoint.clone(),
                    config.realm.clone(),
                ))
            }
            "auth0" => {
                let config = self.auth0.as_ref()
                    .expect("Auth0 config required");
                Arc::new(Auth0UserProvider::new(config.domain.clone()))
            }
            _ => Arc::new(MockUserProvider::new()),
        }
    }
}
```

### 7.2 自定义 Provider

如果需要完全自定义，可以通过配置指定：

```yaml
user_management:
  provider: "custom"
  custom:
    # 自定义 API 端点
    get_user_url: "https://my-idp.com/api/user"
    update_user_url: "https://my-idp.com/api/user"
    headers:
      X-Custom-Auth: "value"
```
