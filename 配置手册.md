## 登录回调链路总览

运维侧需要重点区分两个地址：

1. 网关回调地址（OIDC Provider 使用）：`OIDC_REDIRECT_URL=http://<gateway-host>/auth/callback`
2. 前端回跳地址（frontend callback）：通过 `/auth/login?callback=<frontend_callback_url>` 传入

标准链路：

```
Frontend -> /auth/login?callback=<frontend_callback_url>
OIDC Provider -> /auth/callback?code=<auth_code>&state=<state>   (Gateway)
Gateway -> 302 <frontend_callback_url>?code=success
```

不要把 `OIDC_REDIRECT_URL` 配成前端页面地址（如 `/login/callback`），否则会导致回调中间态落到前端。

## 配置来源

Stew 支持多种配置来源，按以下优先级（低到高）合并：

1. **默认值** -- 代码中定义的回退值
2. **YAML 配置文件** -- `config.yaml`（可选）
3. **环境变量** -- 直接设置或通过 `.env` 文件加载
4. **ETCD 远程配置** -- 最高优先级，覆盖所有本地配置

---

## YAML 配置文件

配置文件路径可通过 `CONFIG_PATH` 环境变量指定，默认查找项目根目录的 `config.yaml`。

### 完整示例

```yaml
# 服务器配置
server:
  host: "127.0.0.1"
  port: 3012
  timeout: "30s"             # 支持 s/m/h/d 单位
  max_connections: 1000
  tls:
    enabled: false
    cert_path: "/app/certs/server.crt"
    key_path: "/app/certs/server.key"
    require_client_auth: false
    client_ca_path: "/app/certs/ca.crt"

# gRPC 后端配置
grpc:
  endpoints:
    - url: "http://127.0.0.1:50051"
      weight: 3
    - url: "http://127.0.0.1:50052"
      weight: 2
    - url: "http://127.0.0.1:50053"
      weight: 1
  descriptors:
    - "/app/proto/descriptor.pb"
    - "/app/proto/stew_descriptor.pb"
  load_balance: "weighted_round_robin"
  pool_size: 10
  timeout: "30s"

# OIDC 认证配置
oidc:
  enabled: false
  client_id: "your-client-id"
  client_secret: "your-client-secret"
  issuer_url: "https://accounts.google.com"
  redirect_url: "http://localhost:3012/auth/callback"  # 必须是网关回调地址，不是前端页面地址
  scopes:
    - "openid"
    - "profile"
    - "email"

# API Key 认证配置
api_key:
  enabled: false
  header_name: "x-api-key"
  validation_mode: "local"       # local | grpc_only
  skip_paths:
    - "/health"
    - "/metrics"
    - "/_openapi"

# 日志配置
logging:
  level: "info"                  # trace | debug | info | warn | error
  format: "pretty"               # pretty | json | compact
  show_file: true
  show_target: false

# 本地服务配置
local_services:
  enabled: true
  routing_strategy: "local_first" # local_first | local_only | remote_only

# 路由冲突检测
route_conflict:
  local_enabled: true
  distributed_enabled: true
  on_conflict: "reject"          # reject | warn | override

# PostgreSQL 持久化
database:
  enabled: false
  url: "postgresql://user:password@localhost:5432/stew_db"
  max_connections: 10
  min_connections: 2
  connect_timeout: "10s"
  query_timeout: "30s"
  cleanup_interval_secs: 3600
  stale_threshold_secs: 86400
```

## 配置项详解

### server -- 服务器

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `host` | String | `"127.0.0.1"` | 绑定地址 |
| `port` | u16 | `3012` | 监听端口 |
| `timeout` | Duration | `30s` | 请求超时 |
| `max_connections` | usize | `1000` | 最大并发连接数 |

### server.tls -- TLS 配置

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用 HTTPS |
| `cert_path` | String? | -- | 服务器证书路径（PEM） |
| `key_path` | String? | -- | 私钥路径（PEM） |
| `require_client_auth` | bool | `false` | 是否启用 mTLS |
| `client_ca_path` | String? | -- | 客户端 CA 证书路径 |

### grpc -- gRPC 后端

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `endpoints` | Vec | 一个 50053 端点 | gRPC 后端端点列表（url + weight） |
| `descriptors` | Vec | descriptor.pb + stew_descriptor.pb | Protobuf 描述符文件路径 |
| `load_balance` | String | `"round_robin"` | 负载均衡策略 |
| `pool_size` | usize | `10` | 每个端点的连接池大小 |
| `timeout` | Duration | `30s` | gRPC 请求超时 |

**负载均衡策略可选值**：

| 值 | 说明 |
|----|------|
| `single` | 单连接（无 LB） |
| `round_robin` | 轮询 |
| `weighted_round_robin` | 加权轮询 |
| `least_connections` | 最少连接 |
| `consistent_hash` | 一致性哈希 |

### oidc -- OIDC 认证

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用 OIDC |
| `client_id` | String | `""` | OIDC Client ID |
| `client_secret` | String | `""` | OIDC Client Secret |
| `issuer_url` | String | `""` | OIDC Provider URL |
| `redirect_url` | String | `""` | OIDC Provider 回调地址（必须是网关 `/auth/callback`） |
| `scopes` | Vec | `["openid", "profile", "email"]` | OAuth Scopes |

**OIDC 回调地址配置约定（强烈建议运维重点核对）**

- `redirect_url` / `OIDC_REDIRECT_URL` 只用于 OIDC Provider 回调，必须配置为网关地址：`http://<gateway-host>/auth/callback`
- 前端回跳页（例如 `/login/callback`）不在网关配置里设置，而是由前端在登录时通过 `GET /auth/login?callback=<frontend_url>` 传入
- 若误将 `redirect_url` 配成前端地址，前端会直接收到原始授权码 `code+state`，导致回调链路异常（常见现象：登录回调报错、偶发重试后恢复）

正确示例：

```yaml
oidc:
  redirect_url: "http://gateway.example.com/auth/callback"
```

错误示例（不要这样配）：

```yaml
oidc:
  redirect_url: "http://frontend.example.com/login/callback"
```

### api_key -- API Key 认证

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用 API Key |
| `header_name` | String | `"x-api-key"` | API Key 请求头名称 |
| `validation_mode` | String | `"local"` | 验证模式（local/grpc_only） |
| `skip_paths` | Vec | `[]` | 跳过认证的路径前缀 |

### logging -- 日志

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `level` | String | `"info"` | 日志级别（trace/debug/info/warn/error） |
| `format` | String | `"pretty"` | 日志格式（pretty/json/compact） |
| `show_file` | bool | `true` | 是否显示文件名和行号 |
| `show_target` | bool | `false` | 是否显示模块路径 |

### local_services -- 本地服务

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `true` | 是否启用本地服务处理 |
| `routing_strategy` | String | `"local_first"` | 路由策略（local_first/local_only/remote_only） |

### route_conflict -- 路由冲突检测

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `local_enabled` | bool | `true` | 启用本地路由冲突检测 |
| `distributed_enabled` | bool | `true` | 启用分布式路由冲突检测（ETCD） |
| `on_conflict` | String | `"reject"` | 冲突处理（reject/warn/override） |

### database -- PostgreSQL 持久化

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用数据库 |
| `url` | String | `""` | PostgreSQL 连接字符串 |
| `max_connections` | u32 | `10` | 最大连接数 |
| `min_connections` | u32 | `2` | 最小连接数 |
| `connect_timeout` | Duration | `10s` | 连接超时 |
| `query_timeout` | Duration | `30s` | 查询超时 |
| `cleanup_interval_secs` | u64 | `3600` | 清理间隔（秒），0 禁用 |
| `stale_threshold_secs` | u64 | `86400` | 过期阈值（秒） |

## 环境变量

以下环境变量可直接设置或通过 `.env` 文件加载（使用 `dotenvy`）。

### 核心变量（直接映射到 `AppConfig::from_env()`）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `LISTEN_ADDR` | `127.0.0.1:3012` | 监听地址（host:port） |
| `GRPC_BACKEND_URL` | -- | 默认 gRPC 后端 URL（可为空） |
| `RUST_ENV` | `development` | 运行环境（development/production） |

### TLS

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `TLS_ENABLED` | `false` | 启用 HTTPS |
| `TLS_CERT_PATH` | -- | 服务器证书路径 |
| `TLS_KEY_PATH` | -- | 私钥路径 |
| `TLS_CLIENT_CA_PATH` | -- | 客户端 CA 路径（mTLS） |
| `TLS_REQUIRE_CLIENT_AUTH` | `false` | 需要客户端证书 |

### ETCD（服务发现/远程配置）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ETCD_ENABLED` | `false` | 启用 ETCD |
| `ETCD_ENDPOINTS` | `http://127.0.0.1:2379` | ETCD 端点（逗号分隔） |
| `ETCD_CONFIG_PREFIX` | `/stew/config` | 远程配置前缀 |

### 认证

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `OIDC_CLIENT_ID` | -- | OIDC Client ID |
| `OIDC_CLIENT_SECRET` | -- | OIDC Client Secret |
| `OIDC_ISSUER_URL` | -- | OIDC Provider URL |
| `OIDC_REDIRECT_URL` | -- | OIDC Provider 回调地址（必须是网关 `/auth/callback`） |
| `AUTH_STRATEGY` | `jwt_first` | 认证策略（jwt_first/api_key_first/jwt_only/api_key_only/either） |

示例：

```bash
# 正确：配置网关回调地址
OIDC_REDIRECT_URL=http://localhost:3012/auth/callback

# 说明：前端回跳页通过 /auth/login?callback=... 传入，不在这里配置
```

### 数据库

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DATABASE_URL` | -- | PostgreSQL 连接字符串 |
| `DATABASE_ENABLED` | `false` | 启用数据库 |

## ETCD 远程配置

启用 ETCD 后，网关会从 `${ETCD_CONFIG_PREFIX}` 前缀下动态读取配置并覆盖本地值。

```bash
# 写入远程配置示例
etcdctl put /stew/config/server/port "8080"
etcdctl put /stew/config/logging/level "debug"
```

远程配置变更通过 watch 机制自动推送，网关收到后热重载。

## 服务路由配置

服务路由可通过 `config/service_routing.yaml` 静态配置：

```yaml
# 默认端点（未匹配的远程服务路由到此）
default_endpoint: "http://127.0.0.1:50053"

# 按服务名静态配置端点
services:
  - service_name: "example.MyService"
    endpoint_url: "http://127.0.0.1:50054"
    weight: 1
    enabled: true
```

注意：本地服务（已在 `LocalServiceRegistry` 注册的）无需配置路由。
