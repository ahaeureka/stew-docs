# 模块详解

> 返回 [README](README.md) | 参阅 [项目架构](项目架构.md)

## 模块总览

```
src/
  lib.rs                    -- crate 入口，模块导出与向后兼容
  main.rs                   -- 应用启动入口
  proto.rs                  -- Protobuf 生成代码引用
  app/                      -- 应用初始化与编排
  core/                     -- 核心业务逻辑
  middleware/               -- 中间件实现
  openapi/                  -- OpenAPI 文档服务
  server/                   -- HTTP/HTTPS 服务器
  config/                   -- 配置管理系统
  discovery/                -- ETCD 服务发现
  auth/                     -- 认证适配器
  dao/                      -- 数据访问层
```

---

## app/ -- 应用初始化与编排

负责将各子系统按依赖关系装配为完整的运行实例。

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `application.rs` | `StewApplication` | 统一装配入口。按序初始化描述符 -> 本地注册表 -> 认证 -> 授权 -> 代理 -> 服务发现 -> 中间件 -> 后台任务。提供 `initialize(config)` 和 `from_env()` 两种创建方式 |
| `descriptor_initializer.rs` | `DescriptorInitializer` | 加载核心服务的 `.pb` 描述符文件（auth/authz/apikey/discovery），设置 `DynamicFirst` 查找模式 |
| `auth_initializer.rs` | `AuthInitializer`, `AuthServices` | 初始化 OIDC 认证、用户服务和 API Key 服务。无数据库时提供 `ApiKeyStubAdapter` 存根 |
| `authz_initializer.rs` | `AuthzInitializer`, `AuthzServices` | 初始化 OPA + Casbin 混合授权引擎、审计日志、策略同步器，注册到本地注册表 |
| `proxy_configurator.rs` | `ProxyConfigurator` | 创建 `ServiceAwareChannelProvider` + `HybridProxy`（LocalFirst 策略），支持 YAML 路由配置 |
| `middleware_configurator.rs` | `MiddlewareConfigurator` | 组装完整的中间件管线（请求 + 响应），严格按序 |
| `service_discovery_initializer.rs` | `ServiceDiscoveryInitializer` | 创建 ETCD 服务发现实例，配置路由冲突检测、健康检查、路由重载回调 |
| `background_tasks.rs` | `BackgroundTaskManager` | 管理后台任务：ETCD 监听、服务状态/健康/恢复监控 |

## core/ -- 核心业务逻辑

核心模块是整个网关的心脏，包含代理、映射、描述符管理、通道选择等关键逻辑。

### 代理

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `hybrid_proxy.rs` | `HybridProxy<C>`, `RoutingStrategy` | 混合代理核心。按 LocalFirst/LocalOnly/RemoteOnly 策略分发请求到本地服务或远程 gRPC |
| `proxy.rs` | `GrpcProxy<C>`, `Proxy` trait, `GrpcProxyTrait` | gRPC 代理实现，支持 Unary/Server Streaming/Client Streaming/Bidi Streaming |
| `local_proxy.rs` | `LocalServiceProxy` | 本地服务代理，将请求路由到注册的本地服务，处理 gRPC Status 到 HTTP 状态码转换 |
| `proxy_common.rs` | `MethodFinder`, `HttpRequestParser`, `ErrorResponseBuilder`, `StreamConverter` | 共享的代理基础设施：方法发现、请求解析、错误构建、流转换 |

### 映射与反射

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `descriptor_manager.rs` | `DescriptorManager`, `ServiceLookupMode` | 管理多个 Protobuf DescriptorPool（主 + 动态），支持 DynamicFirst/MainOnly/DynamicOnly 查找模式 |
| `rest_mapper.rs` | `RouteKey`, `RouteMetadata`, `RouteEntry`, `HttpMethod` | 从 `google.api.http` 注解构建 REST 路由表，支持精确/正则匹配、路径参数、additional_bindings |
| `route_extractor.rs` | `extract_routes_from_descriptor()` | 从 protobuf descriptor 中提取 HTTP 路由信息 |
| `path_param_parser.rs` | `PathParamInjector` | 将 HTTP 路径参数（如 `/users/{id}`）注入到 DynamicMessage，支持嵌套字段 |
| `value_converter.rs` | `ValueConverter` | 字符串/JSON 到 `prost_reflect::Value` 的类型安全转换，覆盖 bool/int/float/enum/bytes/message 等 |

### 通道与负载均衡

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `service_aware_channel_provider.rs` | `ServiceAwareChannelProvider`, `ServiceRoutingEndpoint` | 按服务全名路由到对应端点集群，支持精确匹配 -> 包前缀 -> 默认 Channel |
| `channel_provider.rs` | `ChannelProvider` trait, `SingleChannelProvider`, `PooledChannelProvider` | Channel 抽象接口，支持单连接、连接池、LB 多种实现 |
| `grpc_endpoint.rs` | `GrpcEndpoint` | 单个 gRPC 后端端点封装，含地址/Channel/连接计数/健康状态/权重 |

### 服务注册与发现

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `service_discovery.rs` | `EtcdServiceRegistry`, `EtcdServiceDiscovery`, `ServiceInstanceExt` | ETCD 服务注册（二进制存储/租约/keepalive）和发现（watch/缓存/广播） |
| `service_registry.rs` | `LoadBalanceStrategy`, `ServiceEndpoint`, `ServiceConfig` | 服务端点配置、LB 策略定义 |
| `local_service.rs` | `LocalServiceHandler` trait, `LocalServiceRegistry` | 本地服务处理器接口和注册表 |
| `service_bridge.rs` | `MethodRouter`, `MethodHandler` | 通过闭包自动桥接 tonic 服务方法为 `LocalServiceHandler`，无需手写 adapter |
| `universal_adapter.rs` | `UniversalServiceAdapter` | 字节级通用适配器，不依赖 `ReflectMessage` trait |
| `distributed_route_registry.rs` | `DistributedRouteRegistry` | 基于 ETCD CAS 事务的分布式路由唯一性保证 |
| `health_checker.rs` | `ServiceHealthChecker`, `HealthCheckResult` | 服务健康探测（自定义方法/随机心跳/ETCD lease 刷新） |

### 编解码与工具

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `codec.rs` | `BytesCodec`, `DynamicMessageCodec` | 原始字节和 DynamicMessage 两种 gRPC 编解码 |
| `stream.rs` | `SseHandler`, `WebSocketHandler`, `ClientStreamHandler` | SSE/WebSocket 协议处理，gRPC 流到 HTTP 流桥接 |
| `types.rs` | `SingleChunkBody`, `BoxedBody` | 自定义 HTTP Body 实现 |
| `utils.rs` | `dynamic_to_json()`, `encode_grpc_frame()`, `decode_grpc_frame()` | DynamicMessage<->JSON、gRPC 帧编解码、Timestamp 格式化 |
| `adapter.rs` | `HttpGrpcService` | hyper Service trait 实现，路由 HTTP 到 gRPC proxy，集成中间件/SSE/WebSocket |

### 上下文传递

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `request_context.rs` | `REQUEST_METADATA` (task_local) | 通过 tokio task_local 传递 gRPC MetadataMap，解决本地服务访问 HTTP headers 问题 |
| `request_id_context.rs` | `REQUEST_ID` (task_local) | 全局 request_id 上下文，request 全生命周期可访问 |
| `app_config.rs` | `AppConfig`, `ServerConfig`, `AuthConfig` 等 | 应用配置结构定义（serde 派生） |

## middleware/ -- 中间件实现

全部中间件及其用途，详见 [中间件](中间件.md)。

| 文件 | 关键类型 | 类别 | 功能 |
|------|----------|------|------|
| `request_id.rs` | `RequestIdMiddleware`, `RequestIdSpanMiddleware`, `RequestIdResponseMiddleware` | 请求/响应 | ID 生成/提取、tracing span 绑定、响应头注入 |
| `client_context.rs` | `ClientContextMiddleware` | 请求 | 提取客户端 IP/UA/设备信息，注入 `x-client-context` |
| `unified_auth.rs` | `UnifiedAuthMiddleware`, `AuthStrategy` | 请求 | JWT + API Key 组合认证（5 种策略） |
| `auth.rs` | `AuthMiddleware` | 请求 | JWT 认证（Session Cookie + Bearer Token） |
| `api_key.rs` | `ApiKeyMiddleware` | 请求 | API Key 认证（protobuf 选项控制 + RustAuth 验证） |
| `auth_utils.rs` | `AuthPathChecker`, `AuthHeaderInjector` | 工具 | 认证路径白名单判断、protobuf 选项解析、Header 注入 |
| `http_to_grpc_metadata.rs` | `HttpToGrpcMetadataMiddleware` | 请求 | HTTP 请求头按前缀/映射转为 gRPC metadata |
| `logging.rs` | `LoggingMiddleware` | 请求 | 请求日志记录（method/uri/headers） |
| `cors.rs` | `CorsMiddleware` | 响应 | CORS 跨域头注入，支持 permissive 模式 |
| `grpc_metadata.rs` | `GrpcMetadataMiddleware` | 响应 | gRPC trailers 转 HTTP 响应头 |
| `http_response_wrapper.rs` | `HttpResponseWrapperMiddleware` | 响应 | REST 响应转统一 `stew.api.v1.HttpResponse` 格式 |
| `redirect.rs` | `RedirectResponseMiddleware` | 响应 | 检测 `RedirectResponse` 并转换为 HTTP 3xx 重定向 |
| `header_validation.rs` | `HeaderValidationMiddleware` | 请求 | 验证必需请求头是否存在 |

## auth/ -- 认证适配器

将 `rustauth` 库桥接到 Stew 的本地服务注册表。

| 文件 | 关键类型 | 功能 |
|------|----------|------|
| `adapter_factory.rs` | -- | 适配器工厂（推荐使用） |
| `config.rs` | -- | OIDC 配置管理（已废弃，保留兼容） |
| `token.rs` | -- | JWT 令牌处理（已废弃，保留兼容） |
| `utils.rs` | -- | 工具函数（cookie 设置等） |

> 注意：旧版 `adapter.rs`（`AuthzServiceAdapter`）和 `service.rs`（`AuthzService`）已删除。
> 认证服务现由 `rustauth::services::AuthenticationServiceImpl` 直接提供，
> 通过 `adapter_factory` 注册到本地服务。

| `wrr.rs` | Weighted Round Robin -- 加权轮询 |
| `lc.rs` | Least Connections -- 最少连接 |
| `sed.rs` | Shortest Expected Delay -- 最短期望延迟 |
| `wlc.rs` | Weighted Least Connections -- 加权最少连接 |
| `nq.rs` | Never Queue -- 最快响应 |

| `conn.rs` | 连接管理 |
