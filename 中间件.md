## 中间件管线概述

Stew 的中间件分为**请求中间件**和**响应中间件**两类，分别在请求进入代理之前和响应返回客户端之前执行。

### 执行顺序

```
请求方向（从上到下）：
  RequestIdSpanMiddleware     -- 生成/提取 Request ID，绑定 tracing span
  ClientContextMiddleware     -- 提取客户端信息
  UnifiedAuthMiddleware       -- 统一认证（JWT + API Key）
  HttpToGrpcMetadataMiddleware -- HTTP Headers -> gRPC Metadata
  LoggingMiddleware           -- 请求日志记录
  --------------- HybridProxy 处理请求 ---------------
响应方向（从下到上）：
  GrpcMetadataMiddleware      -- gRPC Trailers -> HTTP Headers
  RequestIdResponseMiddleware -- 注入 Request ID 到响应头
  CorsMiddleware              -- 注入 CORS 跨域头
  HttpResponseWrapperMiddleware -- 统一响应格式包装
  RedirectResponseMiddleware  -- 处理重定向响应
```

中间件链在 `src/app/middleware_configurator.rs` 中配置。

## 请求中间件

### RequestIdSpanMiddleware

**源码**：`src/middleware/request_id.rs`

从请求头中提取 `x-request-id`，如不存在则自动生成 UUID。将 Request ID 绑定到 tracing span，使得整个请求链路的日志都包含该 ID。

- 默认头名称：`x-request-id`（可自定义）
- 生成方式：UUID v4
- 上下文传递：通过 tokio `task_local` 全局可访问

```rust
// 创建中间件
let middleware = RequestIdSpanMiddleware::new();

// 自定义头名称
let middleware = RequestIdSpanMiddleware::new()
    .with_header_name("x-trace-id");
```

### ClientContextMiddleware

**源码**：`src/middleware/client_context.rs`

从 HTTP 请求中提取客户端上下文信息：

| 提取项 | 来源 |
|--------|------|
| 客户端 IP | `X-Forwarded-For` > `X-Real-IP` > 连接地址 |
| User-Agent | `User-Agent` 请求头 |
| 设备/浏览器/OS | 从 UA 解析 |

提取结果序列化为 JSON 并注入 `x-client-context` gRPC metadata。

```rust
let middleware = ClientContextMiddleware::new()
    .with_log_failures(true)
    .with_verbose_logging(false);
```

### UnifiedAuthMiddleware

**源码**：`src/middleware/unified_auth.rs`

编排 JWT 和 API Key 两种认证方式，支持 5 种策略：

| 策略 | 行为 |
|------|------|
| `JwtFirst` | 先尝试 JWT，失败再尝试 API Key |
| `ApiKeyFirst` | 先尝试 API Key，失败再尝试 JWT |
| `JwtOnly` | 仅 JWT 认证 |
| `ApiKeyOnly` | 仅 API Key 认证 |
| `Either` | JWT 或 API Key 任一通过即可 |

通过 protobuf 服务/方法选项控制认证行为：
- 服务级别的 `auth_required` 选项
- 方法级别的 `auth_required` 选项覆盖服务级别
- 白名单路径自动跳过认证

```rust
// JWT 优先策略
let middleware = UnifiedAuthMiddleware::jwt_first(auth_config, apikey_config);

// 仅 JWT
let middleware = UnifiedAuthMiddleware::jwt_only(auth_config);
```

### AuthMiddleware（内部组件）

**源码**：`src/middleware/auth.rs`

JWT 认证核心实现，支持两种认证方式：

1. **Session Cookie** -- 从 Cookie 中获取 session_id -> SessionStore 查找 -> 获取 access_token
2. **Bearer Token** -- 从 `Authorization: Bearer <token>` 头直接获取

验证通过后将用户信息注入请求头（`x-user-id`、`x-user-email` 等）。

### ApiKeyMiddleware（内部组件）

**源码**：`src/middleware/api_key.rs`

API Key 认证，从 `x-api-key` 请求头提取密钥，通过 RustAuth 验证：

- 本地数据库验证（`DATABASE_URL` 已配置时）
- 远程 gRPC 验证

### HttpToGrpcMetadataMiddleware

**源码**：`src/middleware/http_to_grpc_metadata.rs`

将 HTTP 请求头转换为 gRPC Metadata：

```rust
// 转换所有头
let middleware = HttpToGrpcMetadataMiddleware::convert_all();

// 按前缀过滤
let middleware = HttpToGrpcMetadataMiddleware::with_prefixes(vec!["x-custom-"]);

// 自定义映射
let middleware = HttpToGrpcMetadataMiddleware::new()
    .add_mapping("Authorization", "authorization")
    .exclude_headers(vec!["host", "content-type"]);
```

默认排除的头：`host`、`content-type`、`content-length`、`connection`、`transfer-encoding`。

### LoggingMiddleware

**源码**：`src/middleware/logging.rs`

记录请求日志，含 request_id、HTTP method、URI、关键头信息。

### HeaderValidationMiddleware

**源码**：`src/middleware/header_validation.rs`

验证必需的请求头是否存在：

```rust
let middleware = HeaderValidationMiddleware::new(vec![
    "x-api-version".to_string(),
]);
```

## 响应中间件

### GrpcMetadataMiddleware

**源码**：`src/middleware/grpc_metadata.rs`

将 gRPC 响应的 Trailers 转换为 HTTP 响应头：

```rust
// 转换所有 trailers
let middleware = GrpcMetadataMiddleware::convert_all();

// 按前缀过滤
let middleware = GrpcMetadataMiddleware::with_prefixes(vec!["x-"]);

// 排除特定头
let middleware = GrpcMetadataMiddleware::new()
    .exclude_headers(vec!["grpc-status", "grpc-message"]);
```

### RequestIdResponseMiddleware

**源码**：`src/middleware/request_id.rs`

将当前请求的 Request ID 注入到 HTTP 响应头。

### CorsMiddleware

**源码**：`src/middleware/cors.rs`

处理 CORS 跨域请求：

```rust
// 宽松模式（允许所有来源）
let middleware = CorsMiddleware::permissive();

// 自定义配置
let middleware = CorsMiddleware::new()
    .allow_origins(vec!["https://example.com"])
    .allow_methods(vec!["GET", "POST", "PUT", "DELETE"])
    .allow_headers(vec!["Authorization", "Content-Type"]);
```

注入的响应头：
- `Access-Control-Allow-Origin`
- `Access-Control-Allow-Methods`
- `Access-Control-Allow-Headers`
- `Access-Control-Max-Age`

### HttpResponseWrapperMiddleware

**源码**：`src/middleware/http_response_wrapper.rs`

将 REST 响应体转换为统一的 `stew.api.v1.HttpResponse` 格式：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

业务状态码映射（`stew.api.v1.Code`）：

| Code | 含义 | HTTP Status |
|------|------|-------------|
| 0 | Ok | 200 |
| 1 | Unknown | 500 |
| 2 | ParamsError | 400 |
| 3 | AuthError | 401 |
| 4 | NotFound | 404 |
| ... | ... | ... |

### RedirectResponseMiddleware

**源码**：`src/middleware/redirect.rs`

检测响应是否为 `stew.api.v1.RedirectResponse` 类型，如果是则转换为 HTTP 3xx 重定向响应。支持 URL percent-encoding 解码。

## 扩展中间件

### 实现请求中间件

```rust
use stew::core::middleware_interface::{Middleware, MiddlewareError};
use async_trait::async_trait;
use http::Request;
use hyper::body::Incoming;

pub struct MyMiddleware;

#[async_trait]
impl Middleware for MyMiddleware {
    fn name(&self) -> &str {
        "MyMiddleware"
    }

    async fn process(
        &self,
        request: Request<Incoming>,
    ) -> Result<Request<Incoming>, MiddlewareError> {
        // 处理请求...
        Ok(request)
    }
}
```

### 实现响应中间件

```rust
use stew::core::middleware_interface::{ResponseMiddleware, MiddlewareError};
use async_trait::async_trait;
use http::Response;

pub struct MyResponseMiddleware;

#[async_trait]
impl ResponseMiddleware for MyResponseMiddleware {
    fn name(&self) -> &str {
        "MyResponseMiddleware"
    }

    async fn process_response(
        &self,
        response: Response<BoxedBody>,
    ) -> Result<Response<BoxedBody>, MiddlewareError> {
        // 处理响应...
        Ok(response)
    }
}
```

### 注册中间件

在 `src/app/middleware_configurator.rs` 的 `configure()` 方法中添加：

```rust
// 请求中间件（注意顺序）
chain = chain.add(MyMiddleware::new());

// 响应中间件（注意：响应按反向顺序执行）
chain = chain.add_response(MyResponseMiddleware::new());
```

## 中间件接口

### Middleware trait

```rust
#[async_trait]
pub trait Middleware: Send + Sync {
    fn name(&self) -> &str;
    async fn process(
        &self,
        request: Request<Incoming>,
    ) -> Result<Request<Incoming>, MiddlewareError>;
}
```

### ResponseMiddleware trait

```rust
#[async_trait]
pub trait ResponseMiddleware: Send + Sync {
    fn name(&self) -> &str;
    async fn process_response(
        &self,
        response: Response<BoxedBody>,
    ) -> Result<Response<BoxedBody>, MiddlewareError>;
}
```

### MiddlewareError

```rust
pub enum MiddlewareError {
    Unauthorized(String),     // -> 401
    Forbidden(String),        // -> 403
    BadRequest(String),       // -> 400
    InternalError(String),    // -> 500
    Response(Response<...>),  // 直接返回自定义响应
}
```

中间件返回错误时，链条会立即中断（short-circuit），将错误转换为对应 HTTP 响应返回客户端。
