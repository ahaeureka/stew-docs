# 限流（Rate Limiting）

Stew 网关内置了基于 **Redis 固定窗口计数器** 的限流中间件，可对每个 IP 地址和每个认证用户独立施加请求速率上限。

## 工作原理

每个请求到达时，中间件执行以下步骤：

1. 从 `X-Forwarded-For` / `X-Real-IP` 请求头提取客户端 IP。
2. 从 `x-user-id` 请求头提取已认证用户 ID（由认证中间件注入）。
3. 对 **IP 维度**执行计数器检查（始终执行）。
4. 若用户已认证，额外对 **用户 ID 维度**执行计数器检查。
5. 任一维度超限，立即返回 `429 Too Many Requests`。

计数器使用 **1 分钟固定窗口**，通过 Lua 脚本原子地完成 INCR + EXPIRE 两步操作，确保并发安全。

```
每分钟第 0 秒                  第 59 秒  下一分钟第 0 秒
│────────────── 窗口 N ──────────────│────── 窗口 N+1 ──►
```

## 配置

在 `config.yaml`（或 `config.example.yaml`）中添加以下节：

```yaml
rate_limit:
  # 是否启用限流，默认 false
  enabled: true

  # Redis 连接 URL
  redis_url: "redis://127.0.0.1:6379"

  # 每个 IP 地址每分钟的最大请求数
  requests_per_minute: 1000

  # 每个已认证用户每分钟的最大请求数
  # 该限制在 IP 限制通过后额外执行
  requests_per_user_per_minute: 300
```

## 环境变量

也可以通过环境变量覆盖配置（优先级高于 YAML）：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `RATE_LIMIT_ENABLED` | `false` | 是否启用限流 |
| `REDIS_URL` | `redis://127.0.0.1:6379` | Redis 连接 URL |
| `RATE_LIMIT_RPM` | `1000` | 每 IP 每分钟最大请求数 |
| `RATE_LIMIT_USER_RPM` | `300` | 每用户每分钟最大请求数 |

示例（`.env` 文件）：

```env
RATE_LIMIT_ENABLED=true
REDIS_URL=redis://127.0.0.1:6379
RATE_LIMIT_RPM=500
RATE_LIMIT_USER_RPM=200
```

## 中间件位置

限流中间件位于认证中间件**之后**，以便使用认证注入的 `x-user-id` 进行用户级别限流：

```
RequestId
  → ClientContext        ← 提取客户端 IP
    → UnifiedAuth        ← 认证后注入 x-user-id
      → RateLimit        ← 按 IP + 按用户双重限流  (本模块)
        → HttpToGrpcMetadata
          → Logging
```

## 限流维度

### IP 维度（始终执行）

- 标识符：客户端真实 IP（从 `X-Forwarded-For` 或 `X-Real-IP` 提取）
- 配置项：`requests_per_minute`
- 适用对象：所有请求，包括未认证请求

### 用户维度（仅认证请求）

- 标识符：认证中间件注入的 `x-user-id` 请求头
- 配置项：`requests_per_user_per_minute`
- 适用对象：通过 JWT / OIDC Session / API Key 认证的请求
- 两个维度独立计算，单个用户可能同时触发 IP 限制和用户限制

## 响应格式

超限时返回 `HTTP 429 Too Many Requests`，响应体为纯文本说明：

**IP 超限：**
```
HTTP/1.1 429 Too Many Requests
Content-Type: text/plain

Rate limit exceeded: max 1000 requests/min from this IP. Retry after next minute.
```

**用户超限：**
```
HTTP/1.1 429 Too Many Requests
Content-Type: text/plain

Rate limit exceeded: max 300 requests/min for this user. Retry after next minute.
```

> 客户端应在下一分钟开始时重试。下一分钟的开始时间 = `(当前 Unix 时间戳 / 60 + 1) * 60`。

## Redis 键结构

| 键格式 | 示例 | 说明 |
|--------|------|------|
| `stew:rl:ip:{IP}:{window}` | `stew:rl:ip:10.0.0.1:30220130` | IP 维度计数器 |
| `stew:rl:user:{uid}:{window}` | `stew:rl:user:usr_abc123:30220130` | 用户维度计数器 |

其中 `{window}` = `Unix 秒 / 60`（整除），TTL 为 120 秒（覆盖当前窗口和下一个窗口）。

可通过以下命令查看当前计数：

```bash
# 查看所有限流键
redis-cli KEYS "stew:rl:*"

# 查看特定 IP 的当前计数
WINDOW=$(date +%s | awk '{print int($1/60)}')
redis-cli GET "stew:rl:ip:YOUR_IP:$WINDOW"
```

## 容错行为

Redis 不可用时，中间件采用**失败开放（fail-open）**策略：

- 无法建立 Redis 连接 → 记录 `WARN` 日志，**放行请求**
- Lua 脚本执行失败 → 记录 `WARN` 日志，**放行请求**

这确保 Redis 单点故障不会导致网关拒绝所有流量。如果需要强制拒绝（fail-closed），请在配置中关闭限流并在 Redis 恢复前手动控制流量。

## 快速启用

```bash
# 1. 确认 Redis 运行状态
redis-cli ping   # 预期输出: PONG

# 2. 在 .env 中启用限流
echo "RATE_LIMIT_ENABLED=true" >> .env
echo "REDIS_URL=redis://127.0.0.1:6379" >> .env

# 3. 重启网关
cargo run --release
```

或者在 YAML 配置中：

```yaml
rate_limit:
  enabled: true
  redis_url: "redis://127.0.0.1:6379"
  requests_per_minute: 1000
  requests_per_user_per_minute: 300
```
