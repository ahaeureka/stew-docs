# 业务服务配置参数参考

> 适用版本：当前 Management / RegisterWizard 前端页面  
> 对应导出格式：`{service_name}_config.json`

本文档描述管理端前端页面"注册服务"向导（RegisterWizard）和"服务管理"（Management）中所有可配置参数的含义、类型、默认值及取值范围。配置可通过页面表单填写，也可通过 **Import JSON / Export JSON** 功能以 JSON 文件批量导入导出。

---

## 1. 顶层字段

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `service_name` | string | — | 服务的完整 Protobuf 名称，如 `stew.api.v1.OrderService`。注册后不可修改 |
| `version` | string | `"1.0.0"` | 服务实例版本号，语义化版本 |
| `protocol` | string | `"grpc"` | 传输协议：`grpc` / `http` / `tcp` |
| `tls_enabled` | boolean | `false` | 是否启用 TLS 加密传输 |
| `weight` | integer | `1` | 全局负载均衡权重，范围 1–100 |
| `health_endpoint` | string | `""` | 健康检查 HTTP 路径，如 `/health`。空则不主动轮询 |
| `balance_type` | integer | `1` | 负载均衡算法，见下表 |
| `endpoints` | array | — | 后端节点列表，见 [2. Endpoints](#2-endpoints) |
| `tags` | object | `{}` | 键值对标签，用于服务发现过滤，如 `{"env":"prod","region":"us-west"}` |
| `metadata` | object | `{}` | 任意 JSON 附加信息，可包含 `stew_security` 安全配置 |
| `middleware_config` | object | — | 中间件配置，见 [3. middleware_config](#3-middleware_config) |

### balance_type 取值

| 值 | 算法 |
|---|---|
| `1` | Round Robin（轮询） |
| `2` | Weighted Round Robin（加权轮询） |
| `3` | Consistent Hash（一致性哈希） |
| `4` | Least Connections（最少连接） |
| `5` | Shortest Expected Delay（最短预期延迟） |
| `6` | Weighted Least Connections（加权最少连接） |
| `7` | Never Queue（绝不排队） |

## 2. Endpoints

每个 endpoint 代表一个后端节点地址，`endpoints` 是数组，支持多节点。

```json
"endpoints": [
  {
    "address": "10.0.0.1",
    "port": 9090,
    "weight": 1
  }
]
```

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `address` | string | `"localhost"` | 后端 IP 或域名 |
| `port` | integer | `8080` | 端口号，1–65535 |
| `weight` | integer | `1` | 节点相对权重，配合 `balance_type=2`（加权轮询）或 `balance_type=6`（加权最少连接）生效 |

## 3. middleware_config

中间件配置包含限流和四个安全子模块。

### 3.1 限流

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `rate_limit_enabled` | boolean | `true` | 是否启用限流 |
| `rate_limit_rpm` | integer | `0` | 全局每分钟最大请求数，`0` 表示使用网关全局默认值 |
| `rate_limit_user_rpm` | integer | `0` | 单用户每分钟最大请求数，`0` 表示使用网关全局默认值 |

### 3.2 CORS

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `cors_enabled` | boolean | `false` | 是否启用 CORS 跨域中间件（顶层开关，与 `cors.enabled` 等价） |
| `cors.enabled` | boolean | `false` | 同上 |
| `cors.allow_origins` | string[] | `[]` | 允许的来源域名列表，如 `["https://example.com"]`；`["*"]` 表示不限制 |
| `cors.allow_methods` | string[] | `[]` | 允许的 HTTP 方法，如 `["GET","POST","OPTIONS"]` |
| `cors.allow_headers` | string[] | `[]` | 允许的请求头，如 `["Content-Type","Authorization"]` |
| `cors.expose_headers` | string[] | `[]` | 允许浏览器读取的响应头 |
| `cors.allow_credentials` | boolean | `false` | 是否允许携带 Cookie/凭证 |
| `cors.max_age_secs` | integer | `3600` | Preflight 请求结果缓存时间（秒） |

### 3.3 风控（Risk）

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `risk_enabled` | boolean | `false` | 是否启用风控中间件 |
| `risk.enabled` | boolean | `false` | 同上 |
| `risk.mode` | string | `"observe"` | 运行模式：`observe`（仅记录）/ `enforce`（强制拦截） |
| `risk.default_action` | string | `"allow"` | 未命中规则时的默认动作：`allow` / `block` / `challenge` |
| `risk.challenge_paths` | string[] | `[]` | 仅对指定路径前缀执行挑战验证 |
| `risk.block_paths` | string[] | `[]` | 仅对指定路径前缀执行拦截 |
| `risk.observe_only_paths` | string[] | `[]` | 指定路径只记录日志，不执行任何阻断 |
| `risk.high_risk_countries` | string[] | `[]` | 高风险国家/地区代码列表（ISO 3166-1 alpha-2），如 `["CN","RU"]` |
| `risk.challenge_proxy_traffic` | boolean | `false` | 代理流量是否执行挑战验证 |
| `risk.block_datacenter_traffic` | boolean | `false` | 是否拦截数据中心 IP 的流量 |
| `risk.allow_tor_exit_nodes` | boolean | `false` | 是否放行 Tor 出口节点流量 |
| `risk.bot_score_threshold` | number | `0` | Bot 评分阈值（0–1），超过此值触发规则；`0` 表示不限制 |
| `risk.proxy_score_threshold` | number | `0` | 代理评分阈值（0–1）；`0` 表示不限制 |
| `risk.action_overrides` | array | `[]` | 自定义规则覆盖，见 [3.3.1 action_overrides](#331-action_overrides) |

#### 3.3.1 action_overrides

```json
"action_overrides": [
  {
    "name": "block-tor",
    "enabled": true,
    "path_prefixes": ["/api/"],
    "countries": ["CN"],
    "proxy": true,
    "tor": null,
    "datacenter": null,
    "min_bot_score": 0.8,
    "max_bot_score": 1.0,
    "action": "block"
  }
]
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | string | 规则名称（便于识别） |
| `enabled` | boolean | 是否启用此规则 |
| `path_prefixes` | string[] | 匹配的 URL 路径前缀列表 |
| `countries` | string[] | 匹配的国家/地区代码列表 |
| `proxy` | boolean \| null | `true`=代理流量，`false`=非代理，`null`=不限 |
| `tor` | boolean \| null | `true`=Tor 流量，`false`=非 Tor，`null`=不限 |
| `datacenter` | boolean \| null | `true`=数据中心 IP，`false`=非数据中心，`null`=不限 |
| `min_bot_score` | number | Bot 评分下限（0–1） |
| `max_bot_score` | number | Bot 评分上限（0–1） |
| `action` | string | 命中时的动作：`allow` / `block` / `challenge` |

### 3.4 Turnstile 人机验证

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `turnstile_enabled` | boolean | `false` | 是否启用 Cloudflare Turnstile 人机验证 |
| `turnstile.enabled` | boolean | `false` | 同上 |
| `turnstile.required_paths` | string[] | `[]` | 需要验证的路径前缀列表；为空则全局生效 |
| `turnstile.skip_paths` | string[] | `[]` | 跳过验证的路径前缀列表 |
| `turnstile.expected_action` | string | `""` | 期望的 Turnstile action 字段值，为空则不校验 |
| `turnstile.expected_hostname` | string | `""` | 期望的 Turnstile hostname，为空则不校验 |
| `turnstile.enforce_on_risk_challenge` | boolean | `false` | 风控挑战时是否同步要求通过 Turnstile 验证 |

### 3.5 AI Guard（AI 内容防护）

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `ai_guard_enabled` | boolean | `false` | 是否启用 AI Guard |
| `ai_guard.enabled` | boolean | `false` | 同上 |
| `ai_guard.mode` | string | `"observe"` | 运行模式：`observe`（仅记录）/ `enforce`（强制拦截） |
| `ai_guard.include_paths` | string[] | `[]` | 仅对指定路径前缀生效；为空则全局生效 |
| `ai_guard.request_body_max_bytes` | integer | `0` | 最大请求体字节数，`0` 使用默认值 |
| `ai_guard.max_input_tokens` | integer | `0` | 单次请求最大输入 Token 数，`0` 不限制 |
| `ai_guard.max_output_tokens` | integer | `0` | 单次请求最大输出 Token 数，`0` 不限制 |
| `ai_guard.max_context_tokens` | integer | `0` | 最大上下文 Token 数（含历史），`0` 不限制 |
| `ai_guard.history_policy` | string | `"truncate_to_max_tokens"` | 超长历史处理策略：`truncate_to_max_tokens` / `drop_oldest` / `reject` |
| `ai_guard.daily_token_quota` | integer | `0` | 每日 Token 配额（用户级），`0` 不限制 |
| `ai_guard.daily_request_quota` | integer | `0` | 每日请求次数配额（用户级），`0` 不限制 |
| `ai_guard.minute_request_quota` | integer | `0` | 每分钟请求次数配额（用户级），`0` 不限制 |
| `ai_guard.quota_window_secs` | integer | `86400` | 配额统计窗口长度（秒），默认 24 小时 |
| `ai_guard.allow_free_chat` | boolean | `true` | 是否允许非 AI 类请求（非 `/chat/completions` 等）通过 |
| `ai_guard.allowed_topics` | string[] | `[]` | 话题白名单列表，不在列表中的话题会被阻断（列表为空则不限制） |
| `ai_guard.deny_keywords` | string[] | `[]` | 关键词黑名单，请求体或响应体包含时触发规则 |
| `ai_guard.enable_audit` | boolean | `true` | 是否记录审计日志 |
| `ai_guard.classifier_type` | string | `"rule"` | 话题分类器类型：`rule`（规则）/ `llm`（LLM 辅助） |
| `ai_guard.llm_endpoint` | string | `""` | LLM 分类器接口地址（仅 `classifier_type=llm` 时生效） |
| `ai_guard.llm_model` | string | `""` | LLM 分类器使用的模型名 |
| `ai_guard.llm_system_prompt` | string | `""` | LLM 分类器的系统提示词 |
| `ai_guard.llm_timeout_ms` | integer | `5000` | LLM 分类器请求超时（毫秒） |
| `ai_guard.llm_confidence_threshold` | number | `0.7` | LLM 分类置信度阈值（0–1），低于此值时降级为规则分类器 |
| `ai_guard.endpoint_overrides` | array | `[]` | 各 AI 端点的个性化覆盖配置，见 [3.5.1 endpoint_overrides](#351-endpoint_overrides) |

#### 3.5.1 ai_guard.endpoint_overrides

针对不同的 AI API 端点（如 `/v1/chat/completions`、`/v1/embeddings`）设置独立的限额和策略，覆盖全局 `ai_guard` 配置。

```json
"endpoint_overrides": [
  {
    "endpoint_id": "chat",
    "exact_paths": ["/v1/chat/completions"],
    "prefix_paths": [],
    "disabled": false,
    "mode": "enforce",
    "max_input_tokens": 4096,
    "max_output_tokens": 2048,
    "daily_token_quota": 100000,
    "enable_audit": true
  }
]
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `endpoint_id` | string | 端点标识符（必填） |
| `exact_paths` | string[] | 精确匹配的路径列表 |
| `prefix_paths` | string[] | 前缀匹配的路径列表 |
| `disabled` | boolean | 是否对此端点禁用 AI Guard |
| 其余字段 | — | 与 `ai_guard` 顶层字段含义相同，仅对本端点生效 |

## 5. 导入/导出操作说明

### 从 RegisterWizard 导出

1. 进入 **注册服务** 向导第 3 步（Service config）
2. 填写或调整好相关配置
3. 点击右上角 **Export JSON** 按钮
4. 浏览器自动下载 `{service_name}_config.json`

### 从 Management 导出

- **单实例导出**：Overview Tab → 展开服务行 → 点击实例的 **Export** 按钮
- **服务全量导出**：Overview Tab → 服务行 Actions → **Export** 按钮（含该服务所有实例）

### 导入配置

- **RegisterWizard**：在第 3 步点击 **Import JSON**，选择之前导出的 `.json` 文件，表单自动填充
- **Management 跳转注册**：顶部 **Import config** 按钮，选择文件后自动跳转到注册向导并预填配置

> **注意**：`service_name` 字段导入时不会覆盖向导第 1 步已填写的服务名；`protocol` 由第 1 步选择决定。导入的 JSON 可以是单服务配置对象，也可以是批量导出的 `{ instances: [...] }` 格式，导入时取第一个实例的配置。

## 6. 字段兼容性说明

| 场景 | 说明 |
|---|---|
| `cors_enabled` 与 `cors.enabled` | 两者同义，导入时任意一个为 `true` 即生效 |
| `risk_enabled` 与 `risk.enabled` | 同上 |
| `turnstile_enabled` 与 `turnstile.enabled` | 同上 |
| `ai_guard_enabled` 与 `ai_guard.enabled` | 同上 |
| 禁用状态的子配置 | 即使 `xxx_enabled=false`，子配置对象仍完整导出，便于预设待用配置后快速切换开关 |
| `rate_limit_rpm=0` | 值为 `0` 时表示使用网关全局默认限流值，而非"不限制" |
| 配额字段为 `0` | AI Guard 的所有配额字段值为 `0` 时均表示"不限制" |
