## 一、公共说明

### 认证

所有接口均需携带认证头，支持以下任意一种：

| 认证方式 | Header / Metadata |
|----------|-------------------|
| API Key | `x-api-key: <your_api_key>` |
| JWT Bearer | `authorization: Bearer <token>` |

### HTTP 端点基础路径

```
http[s]://<gateway_host>:<port>/api/v1/discovery/
```

### gRPC 服务完整名称

```
stew.api.v1.ServiceDiscoveryService
```

## 二、RegisterService — 注册服务实例

**gRPC 方法**：`RegisterService`  
**HTTP**：`POST /api/v1/discovery/services/register`

### 2.1 请求参数 `RegisterServiceRequest`

| 字段 | JSON Key | 类型 | 必填 | 默认值 | 说明 |
|------|----------|------|------|--------|------|
| `service` | `service` | `ServiceInstance` | 是 | — | 服务实例信息，见 [ServiceInstance](#serviceinstance) |
| `ttl` | `ttl` | `uint32` | 否 | `60` | 注册 TTL（秒），超时未续约则自动注销 |

#### ServiceInstance

| 字段 | JSON Key | 类型 | 必填 | 默认值 | 说明 |
|------|----------|------|------|--------|------|
| `service_name` | `service_name` | `string` | 是 | — | 服务完整名称，如 `stew.api.v1.UserService` |
| `instance_id` | `instance_id` | `string` | 否 | 自动生成 | 实例唯一标识，同服务下不重复；留空则由网关生成 |
| `lb` | `lb` | `LoadBalancer` | 是 | — | 负载均衡配置，含端点列表，见 [LoadBalancer](#loadbalancer) |
| `version` | `version` | `string` | 否 | `1.0.0` | 服务版本号，用于标识及灰度 |
| `metadata` | `metadata` | `Struct (JSON object)` | 否 | `{}` | 任意 KV 元数据，可用于路由过滤 |
| `health_endpoint` | `health_endpoint` | `string` | 否 | — | 健康检查路径，如 `/health` |
| `health_check_config` | `health_check_config` | `HealthCheckConfig` | 否 | — | 健康检查详细配置，见 [HealthCheckConfig](#healthcheckconfig) |
| `status` | `status` | `ServiceStatus (enum)` | 否 | `HEALTHY` | 初始状态，见 [ServiceStatus](#servicestatus) |
| `weight` | `weight` | `uint32` | 否 | `1` | 负载均衡权重，越大分配流量越多 |
| `tags` | `tags` | `map<string, string>` | 否 | `{}` | 服务标签，支持按标签过滤路由 |
| `protocol` | `protocol` | `string` | 否 | `grpc` | 协议类型：`grpc` / `http` / `tcp` |
| `tls_enabled` | `tls_enabled` | `bool` | 否 | `false` | 是否启用 TLS |
| `protobuf_descriptor` | `protobuf_descriptor` | `bytes (base64)` | 否 | — | 编译好的 `.pb` 文件二进制内容，上传后网关自动加载路由 |
| `middleware_config` | `middleware_config` | `ServiceMiddlewareConfig` | 否 | — | 服务级中间件开关配置，见 [ServiceMiddlewareConfig](#servicemiddlewareconfig) |

### 2.2 响应参数 `RegisterServiceResponse`

| 字段 | JSON Key | 类型 | 说明 |
|------|----------|------|------|
| `success` | `success` | `bool` | 是否注册成功 |
| `message` | `message` | `string` | 状态描述 |
| `lease_id` | `lease_id` | `string` | ETCD 租约 ID，用于续约心跳 |

## 五、UploadProtobufDescriptor — 上传描述符

**gRPC 方法**：`UploadProtobufDescriptor`  
**HTTP**：`POST /api/v1/discovery/descriptors/{service_name}/upload`

### 5.1 请求参数 `UploadProtobufDescriptorRequest`

| 字段 | JSON Key | 类型 | 必填 | 默认值 | 说明 |
|------|----------|------|------|--------|------|
| `service_name` | `service_name` | `string` | 是 | — | 服务完整名称，必须与 `.pb` 中的 service package 前缀匹配 |
| `descriptor_data` | `descriptor_data` | `bytes (base64)` | 是 | — | `protoc --descriptor_set_out` 编译的二进制数据，JSON 传输时使用 base64 编码 |
| `descriptor_version` | `descriptor_version` | `string` | 否 | 自动生成 | 版本标识，留空则自动生成 `{timestamp}-{hash_prefix}` |
| `description` | `description` | `string` | 否 | — | 版本描述，便于回滚时识别 |
| `signature` | `signature` | `string` | 否 | — | `HMAC-SHA256(descriptor_data, shared_secret)`，可选完整性校验 |
| `force` | `force` | `bool` | 否 | `false` | `true` 则忽略兼容性警告强制更新 |
| `previous_version` | `previous_version` | `string` | 否 | — | 乐观锁：若当前 active 版本与此值不符则拒绝更新，防止并发冲突 |

> **幂等性**：若 `descriptor_data` 的 SHA-256 与当前 active 版本相同，网关直接返回 `success=true`，不触发热加载。

### 5.2 响应参数 `UploadProtobufDescriptorResponse`

| 字段 | JSON Key | 类型 | 说明 |
|------|----------|------|------|
| `success` | `success` | `bool` | 是否上传成功 |
| `message` | `message` | `string` | 状态描述 |
| `descriptor_key` | `descriptor_key` | `string` | ETCD 存储路径，如 `/descriptors/my-service/v/20260318120000-a1b2c3` |
| `discovered_services` | `discovered_services` | `string[]` | 从描述符中解析出的服务名称列表 |
| `compatibility_warnings` | `compatibility_warnings` | `string[]` | 检测到的兼容性警告（字段删除/类型变更等），不为空时建议检查 |
| `applied_version` | `applied_version` | `string` | 实际生效的版本号（可能与请求中不同） |
| `descriptor_hash` | `descriptor_hash` | `string` | 描述符内容的 SHA-256 哈希（hex 字符串） |

### 5.3 错误码说明

| gRPC Status | HTTP Status | 场景 |
|-------------|-------------|------|
| `INVALID_ARGUMENT` | 400 | 描述符格式错误、服务名不匹配、超过 10MB 限制 |
| `FAILED_PRECONDITION` | 412 | 乐观锁冲突（`previous_version` 与实际不符） |
| `UNAUTHENTICATED` | 401 | 缺少或无效的 API Key |
| `PERMISSION_DENIED` | 403 | API Key 无 `service:descriptor:write` scope |
| `DEADLINE_EXCEEDED` | 504 | ETCD 写入超时 |

## 八、嵌套对象定义

### LoadBalancer

| 字段 | JSON Key | 类型 | 必填 | 说明 |
|------|----------|------|------|------|
| `type` | `type` | `BalanceType (enum)` | 是 | 负载均衡算法，见 [BalanceType](#balancetype) |
| `endpoints` | `parameters` | `Endpoint[]` | 是 | 后端端点列表，至少一个 |

### Endpoint

| 字段 | JSON Key | 类型 | 必填 | 默认值 | 说明 |
|------|----------|------|------|--------|------|
| `address` | `address` | `string` | 是 | — | 后端 IP 或主机名 |
| `port` | `port` | `uint32` | 是 | — | 后端端口 |
| `weight` | `weight` | `uint32` | 否 | `1` | 流量权重，用于 WRR/WLC 算法 |

### HealthCheckConfig

| 字段 | JSON Key | 类型 | 必填 | 默认值 | 说明 |
|------|----------|------|------|--------|------|
| `enabled` | `enabled` | `bool` | 否 | `true` | 是否启用主动健康检查 |
| `grpc_method` | `grpc_method` | `string` | 否 | 自动探测 | gRPC 健康探测方法，如 `stew.api.v1.UserService/GetUser`，空则自动选第一个方法 |
| `http_path` | `http_path` | `string` | 否 | — | HTTP 健康检查路径，如 `/health`，与 `grpc_method` 二选一 |
| `interval_seconds` | `interval_seconds` | `uint32` | 否 | `30` | 探测间隔（秒） |
| `timeout_seconds` | `timeout_seconds` | `uint32` | 否 | `5` | 单次探测超时（秒） |
| `healthy_threshold` | `healthy_threshold` | `uint32` | 否 | `2` | 连续成功 N 次后标记为健康 |
| `unhealthy_threshold` | `unhealthy_threshold` | `uint32` | 否 | `3` | 连续失败 N 次后标记为不健康 |

### ServiceMiddlewareConfig

| 字段 | JSON Key | 类型 | 必填 | 默认值 | 说明 |
|------|----------|------|------|--------|------|

| `rate_limit_rpm` | `rate_limit_rpm` | `uint32` | 否 | `0` | 服务级每分钟请求上限，`0` 沿用全局配置 |
| `rate_limit_user_rpm` | `rate_limit_user_rpm` | `uint32` | 否 | `0` | 单用户每分钟请求上限，`0` 沿用全局配置 |
| `cors_enabled` | `cors_enabled` | `bool` | 否 | `false` | 是否启用 CORS |
| `cors` | `cors` | `ServiceCorsConfig` | 否 | — | CORS 详细配置，见下表 |
| `risk_enabled` | `risk_enabled` | `bool` | 否 | `false` | 是否启用风险评估 |
| `risk` | `risk` | `ServiceRiskConfig` | 否 | — | 风险评估配置 |
| `turnstile_enabled` | `turnstile_enabled` | `bool` | 否 | `false` | 是否启用 Cloudflare Turnstile 验证 |
| `turnstile` | `turnstile` | `ServiceTurnstileConfig` | 否 | — | Turnstile 配置 |

#### ServiceCorsConfig

| 字段 | JSON Key | 类型 | 默认值 | 说明 |

| `allow_methods` | `allow_methods` | `string[]` | `[]` | 允许的 HTTP 方法 |

## 九、枚举值参考

### BalanceType

| 枚举值 | 整数值 | 说明 |
|--------|--------|------|
| `BALANCE_TYPE_UNKNOWN` | `0` | 未指定（不推荐） |
| `BALANCE_TYPE_ROUND_ROBIN` | `1` | 轮询（RR），适合同构后端 |
| `BALANCE_TYPE_WEIGHTED_ROUND_ROBIN` | `2` | 加权轮询（WRR），结合 `weight` 字段 |
| `BALANCE_TYPE_CONSISTENT_HASH` | `3` | 一致性哈希，会话粘滞场景 |
| `BALANCE_TYPE_LEAST_CONNECTIONS` | `4` | 最小连接数（LC） |
| `BALANCE_TYPE_SED` | `5` | 最短预期延迟（SED） |
| `BALANCE_TYPE_WEIGHTED_LEAST_CONNECTIONS` | `6` | 加权最小连接数（WLC） |
| `BALANCE_TYPE_NEVER_QUEUE` | `7` | 永不排队（NQ） |

### ServiceStatus

| 枚举值 | 整数值 | 说明 |
|--------|--------|------|
| `SERVICE_STATUS_UNKNOWN` | `0` | 未知 |
| `SERVICE_STATUS_HEALTHY` | `1` | 健康，参与流量分配 |
| `SERVICE_STATUS_UNHEALTHY` | `2` | 不健康，暂停流量分配 |
| `SERVICE_STATUS_MAINTENANCE` | `3` | 维护模式，停止接收新请求 |
| `SERVICE_STATUS_DRAINING` | `4` | 流量耗尽中，等待现有请求处理完毕 |

## 十、完整请求示例

### 10.1 RegisterService — 最小化注册（无描述符）

```json
POST /api/v1/discovery/services/register
x-api-key: svc_xxxxxxxxxxxx

{
  "service": {
    "service_name": "stew.api.v1.UserService",
    "instance_id": "user-svc-pod-abc123",
    "lb": {
      "type": "BALANCE_TYPE_ROUND_ROBIN",
      "parameters": [
        { "address": "10.0.0.5", "port": 50051 }
      ]
    },
    "version": "v1.2.3",
    "tags": { "env": "prod", "region": "cn-north-1" },
    "protocol": "grpc"
  },
  "ttl": 60
}
```

**响应**：

```json
{
  "success": true,
  "message": "Service registered successfully",
  "lease_id": "7587873756604589057"
}
```
