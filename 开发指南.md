### 构建命令

```bash
# Debug 构建（开发用，含调试信息）
cargo build

# Release 构建（部署用，含优化）
cargo build --release

# 仅构建主项目
cargo build -p stew

# 构建子工作区
cargo build -p rust-loadblance
cargo build -p rustauth
```

### Proto 变更后

如果修改了 `.proto` 文件，需要重新编译：

```bash
# 生成描述符文件
./scripts/generate_descriptor.sh

# 或手动
protoc \
  --descriptor_set_out=proto/descriptor.pb \
  --include_imports \
  --proto_path=proto \
  proto/*.proto
```

`build.rs` 会在 `cargo build` 时自动编译 proto 并生成 Rust 代码。

## 代码规范

### Rust 风格

- 遵循 Rust 2021 edition 规范
- 最大行宽 100 字符
- 使用 `Arc<T>` + `RwLock` 处理并发共享状态
- 优先 `pub(crate)` 可见性
- 避免 unsafe 代码
- 注释和文档使用英文

### 格式化与检查

```bash
# 格式化
cargo fmt --all

# Lint 检查
cargo clippy --all-targets --all-features

# 安全审计
cargo deny check
```

### 提交前检查清单

1. `cargo fmt --all` -- 格式化
2. `cargo clippy --all-targets --all-features` -- 静态检查
3. `cargo test` -- 单元测试通过
4. `cargo check` -- 编译通过

## Protobuf 管理

### 目录结构

```
proto/
  google/                   -- Google API 基础 proto
    api/
      http.proto
      annotations.proto
  stew/
    api/
      v1/
        options.proto       -- 自定义扩展选项（auth_required 等）
  helloworld.proto          -- 示例服务
  authentication.proto      -- 认证服务
  authorization.proto       -- 授权服务
  user.proto                -- 用户服务
  apikey.proto              -- API Key 服务
  service_discovery.proto   -- 服务发现
```

### 编译顺序

`build.rs` 按依赖关系编译（顺序重要）：

1. Google API 基础（`google/api/http.proto`、`annotations.proto`）
2. 自定义扩展（`stew/api/v1/options.proto`）
3. 业务 Proto（`user.proto`、`authentication.proto` 等）

### 重要约定

- 所有业务 proto 使用 `package stew.api.v1`
- 绑定 REST 路由需添加 `google.api.http` 注解

### 添加新 Proto 示例

```protobuf
syntax = "proto3";
package stew.api.v1;

import "google/api/annotations.proto";
import "stew/api/v1/options.proto";

service MyNewService {
  // 可选：服务级认证配置
  option (stew.api.v1.service_auth) = { auth_required: true };

  rpc GetItem(GetItemRequest) returns (GetItemResponse) {
    option (google.api.http) = {
      get: "/api/v1/items/{id}"
    };
  }

  rpc CreateItem(CreateItemRequest) returns (CreateItemResponse) {
    option (google.api.http) = {
      post: "/api/v1/items"
      body: "*"
    };
  }

  rpc ListItems(ListItemsRequest) returns (ListItemsResponse) {
    option (google.api.http) = {
      get: "/api/v1/items"
      additional_bindings {
        get: "/api/v1/items/search"
      }
    };
  }
}
```

## 新增远程服务

推荐方式：通过 ETCD 动态注册。

### 步骤

1. **编写 Proto 定义**，添加 `google.api.http` 注解
2. **编译为 FileDescriptorSet**：
   ```bash
   protoc \
     --descriptor_set_out=my_service.pb \
     --include_imports \
     --proto_path=proto \
     proto/my_service.proto
   ```
3. **注册到 ETCD**（携带描述符）：
   ```bash
   grpcurl -plaintext \
     -d "{
       \"service_name\": \"stew.api.v1.MyNewService\",
       \"host\": \"10.0.1.1\",
       \"port\": 50051,
       \"lb\": \"ROUND_ROBIN\",
       \"protobuf_descriptor\": \"$(base64 -w0 my_service.pb)\"
     }" \
     localhost:3012 stew.api.v1.ServiceDiscoveryService/RegisterService
   ```
4. 网关自动加载描述符并生成 REST 路由

## 新增本地服务

适用于低延迟直连场景。

### 步骤

1. **实现 LocalServiceHandler trait**：

```rust
use stew::core::local_service::LocalServiceHandler;
use async_trait::async_trait;
use prost_reflect::DynamicMessage;
use tonic::metadata::MetadataMap;

pub struct MyServiceHandler;

#[async_trait]
impl LocalServiceHandler for MyServiceHandler {
    async fn handle_unary(
        &self,
        method_name: &str,
        request: DynamicMessage,
        metadata: MetadataMap,
    ) -> Result<DynamicMessage, tonic::Status> {
        match method_name {
            "GetItem" => {
                // 处理请求...
                Ok(response_message)
            }
            _ => Err(tonic::Status::unimplemented(
                format!("Method {} not implemented", method_name)
            )),
        }
    }
}
```

2. **注册到 LocalServiceRegistry**：

```rust
let handler = Arc::new(MyServiceHandler::new());
local_registry.register_service(
    "stew.api.v1.MyNewService",
    handler
).await;
```

3. **使用 UniversalServiceAdapter 简化**（推荐）：

```rust
use stew::core::universal_adapter::UniversalServiceAdapter;

let adapter = UniversalServiceAdapter::builder()
    .unary("GetItem", |req: GetItemRequest| async move {
        let response = GetItemResponse { /* ... */ };
        Ok(response)
    })
    .build();

local_registry.register_service(
    "stew.api.v1.MyNewService",
    Arc::new(adapter)
).await;
```

4. **使用 ServiceBridge 自动桥接**（最简洁）：

```rust
use stew::core::service_bridge::MethodRouter;

let router = MethodRouter::new()
    .register("GetItem", my_service.clone(), |svc, req| {
        Box::pin(async move { svc.get_item(req).await })
    });
```

## 新增中间件

### 步骤

1. **创建中间件文件** `src/middleware/my_middleware.rs`
2. **实现 Middleware 或 ResponseMiddleware trait**（参见 [中间件](中间件.md)）
3. **在 mod.rs 中注册**：
   ```rust
   pub mod my_middleware;
   pub use my_middleware::MyMiddleware;
   ```
4. **在 middleware_configurator.rs 中挂载**：
   ```rust
   chain = chain.add(MyMiddleware::new());
   ```

### 注意事项

- 请求中间件的顺序很重要（认证应在业务逻辑前）
- 响应中间件按反向顺序执行
- 中间件可以 short-circuit（返回 `MiddlewareError` 中断链条）
- 中间件应当是无状态的（或使用 `Arc<RwLock<>>` 管理状态）

## 配置系统扩展

### 添加新的配置源

实现 `ConfigSource` trait：

```rust
#[async_trait]
pub trait ConfigSource: Send + Sync {
    async fn load(&self) -> Result<serde_yaml::Value, ConfigError>;
    fn priority(&self) -> u32;  // 数字越大优先级越高
    fn name(&self) -> &str;
}
```

### 添加新的配置项

1. 在 `src/config/models.rs` 中添加字段：
   ```rust
   pub struct AppConfig {
       // ...existing fields...
       #[serde(default)]
       pub my_feature: MyFeatureConfig,
   }
   ```
2. 在 `src/core/app_config.rs` 中同步（如果使用 `AppConfig::from_env()`）

## 工作区成员

| 成员 | 路径 | 说明 |
|------|------|------|
| `stew` | `.` (根目录) | 主网关项目 |

```toml
[dependencies]
rust-loadblance = { path = "rust-loadblance" }
rustauth = { path = "rustauth" }
```
