# 业务端用户信息查看与更新方案

> 基于 Casdoor OIDC 集成的用户管理功能实现指南

---

## 背景

当前网关已实现：
- ✅ OIDC 登录认证（Session / Bearer Token）
- ✅ 用户信息获取（`GET /auth/user`）
- ❌ 用户信息更新（未实现）

本文档定义如何为业务端提供完整的用户信息管理能力。

## 一、架构设计

### 1.1 整体流程

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐     ┌─────────────┐
│  前端/业务端  │────▶│  Stew 网关    │────▶│ Casdoor    │────▶│  用户数据   │
│             │     │              │     │  OIDC API   │     │  存储       │
└─────────────┘     └──────────────┘     └─────────────┘     └─────────────┘
                           │
                    ┌──────┴──────┐
                    │ UserService  │
                    │ - Get        │
                    │ - Update     │
                    └─────────────┘
```

### 1.2 数据流说明

| 操作 | 数据流 | 说明 |
|------|--------|------|
| 查看用户信息 | 网关 → Casdoor `/api/userinfo` | OIDC 标准 UserInfo endpoint |
| 更新用户信息 | 网关 → Casdoor `/api/update-user` | 需要 access_token 调用管理 API |

```json
  "locale": "zh-CN",

  "updatedAt": 1705123456

```

#### 更新用户信息

**请求**:

**响应**:

## 三、Casdoor API 集成

### 3.1 可用 Casdoor API

根据 Casdoor 文档，关键 API 端点：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/.well-known/openid-configuration` | GET | 获取 OIDC 元数据 |
| `/api/userinfo` | GET | 获取用户信息（需 scope） |
| `/api/update-user` | POST | 更新用户信息 |
| `/api/upload-avatar` | POST | 上传头像 |

### 3.2 Scope 配置

在 OIDC 登录时需要请求以下 scope：

```yaml
oidc:
  scopes:
    - "openid"      # 必须
    - "profile"     # 允许获取用户基本信息
    - "email"       # 允许获取邮箱
    - "phone"       # 允许获取手机号
    - "offline_access"  # 允许刷新 token
```

### 3.3 调用示例

```rust
// 更新用户信息
let client = reqwest::Client::new();
let response = client
    .post(format!("{}/api/update-user", casdoor_url))
    .header("Authorization", format!("Bearer {}", access_token))
    .json(&user_update_payload)
    .send()
    .await?;
```

## 四、实现方案

### 4.1 修改 proto 定义

**文件**: `proto/user.proto`

```protobuf
service UserService {
  option (stew.api.v1.auth_reqiured) = true;
  option (stew.api.v1.http_response) = "stew.api.v1.HttpResponse";

  // 获取当前用户信息
  rpc Get(google.protobuf.Empty) returns (User) {
    option (google.api.http) = {
      get: "/auth/user"
    };
  }

  // 更新当前用户信息
  rpc Update(UpdateUserRequest) returns (User) {
    option (google.api.http) = {
      put: "/auth/user"
      body: "*"
    };
  }
}

// 更新用户请求
message UpdateUserRequest {
  string display_name = 1;
  string preferred_username = 2;
  string locale = 3;
  string zoneinfo = 4;
  string website = 5;
  string picture = 6;
}
```

### 4.2 修改 UserService 实现

**文件**: `rustauth/src/services/user.rs`

新增 `update` 方法：

```rust
#[tonic::async_trait]
impl UserService for UserServiceImpl {
    /// 获取当前用户信息
    async fn get(&self, request: Request<()>) -> Result<Response<User>, Status> {
        // ... 现有实现
    }

    /// 更新当前用户信息
    async fn update(
        &self,
        request: Request<UpdateUserRequest>,
    ) -> Result<Response<User>, Status> {
        let req = request.get_ref();
        let token = self
            .get_auth_from_request(&request)
            .ok_or_else(|| Status::unauthenticated("Missing authorization header"))?;

        // 调用 Casdoor API 更新用户
        let updated_user = self.update_user_in_casdoor(&token, req).await?;

        Ok(Response::new(updated_user))
    }
}
```

### 4.3 调用 Casdoor API

在 `UserServiceImpl` 中添加：

```rust
async fn update_user_in_casdoor(
    &self,
    token: &str,
    req: &UpdateUserRequest,
) -> Result<User, Status> {
    // 获取 Casdoor URL
    let casdoor_url = &self.oidc_config.issuer_url;

    let client = reqwest::Client::new();
    let response = client
        .post(format!("{}/api/update-user", casdoor_url))
        .header("Authorization", format!("Bearer {}", token))
        .json(req)
        .send()
        .await
        .map_err(|e| Status::internal(format!("Failed to update user: {}", e)))?;

    if !response.status().is_success() {
        return Err(Status::internal("Failed to update user in Casdoor"));
    }

    // 解析响应并转换为 User proto
    let user_info: UserInfo = response.json().await
        .map_err(|e| Status::internal(format!("Failed to parse response: {}", e)))?;

    Ok(self.user_info_to_proto(&user_info))
}
```

## 六、安全考虑

### 6.1 权限控制

- 用户只能更新自己的信息（通过 token 验证身份）
- 敏感字段（如 email、phone）可能需要额外验证
- 管理员更新其他用户需单独授权

### 6.2 字段限制

| 字段 | 可更新 | 说明 |
|------|--------|------|
| `displayName` | ✅ | 显示名称 |
| `preferredUsername` | ✅ | 用户名（需唯一性检查） |
| `locale` | ✅ | 语言偏好 |
| `zoneinfo` | ✅ | 时区 |
| `email` | ⚠️ | 需邮箱验证 |
| `phone` | ⚠️ | 需短信验证 |
| `password` | ❌ | 需通过专门流程 |
